# 產品需求文件 (PRD) - 個人記帳簿系統

## 1. 專案概述
- **背景與動機**：現代人生活忙碌，常常不清楚資金的流向。市面上的記帳工具若過於複雜會降低持續使用的意願。本系統旨在提供一個輕量、簡單且直觀的記帳工具，幫助使用者輕鬆培養記帳習慣，進而有效管理個人財務。
- **目標用戶**：小資族/學生、財務規劃者、記帳新手
- **核心價值主張**：提供最直覺的操作介面以減少記帳阻力，並透過清晰的圖表與預算追蹤功能，幫助用戶輕鬆掌握收支流向，達到財務健康。

## 2. 功能需求
1. **快速收支記錄**
   - 作為 **記帳新手**，我希望 **能在一個簡單的介面上快速輸入金額、選擇類別與日期**，以便 **在消費當下立即記帳而不覺得麻煩**。
2. **餘額即時統計**
   - 作為 **小資族**，我希望 **首頁能直接看到本月的總支出、總收入及剩餘可用餘額**，以便 **隨時掌握自己還有多少錢可以使用**。
3. **週期性分析圖表**
   - 作為 **財務規劃者**，我希望 **能以圓餅圖或長條圖等視覺化圖表查看按月或週的收支比例**，以便 **分析我的消費習慣並找出可以節省的開銷**。
4. **自定義類別管理**
   - 作為 **注重細節的用戶**，我希望 **能自己新增、修改或刪除收支類別（如自訂「手搖飲」、「健身」等類別）**，以便 **讓記帳紀錄更貼近我真實的生活習慣**。
5. **預算超支提醒**
   - 作為 **學生**，我希望 **能設定每個月的支出預算，當花費接近或超過設定的預算時能獲得視覺化的警告提醒**，以便 **及時控制消費慾望、避免透支**。

## 3. 非功能需求
- **技術限制**：系統預計使用 Flask 框架搭配 Jinja2 模板，後端資料庫以 SQLite 進行儲存。
- **效能考量**：頁面載入速度需小於 2 秒，資料讀寫需即時響應。前端介面需具備 RWD（響應式網頁設計），確保在手機及電腦設備上都有良好的視覺體驗。
- **安全考量**：防止 SQL Injection 與 XSS 等常見的注入攻擊，確保用戶輸入與資料庫的安全性。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**
  - 快速收支記錄（新增、讀取、編輯、刪除收支明細）
  - 餘額即時統計（明細列表與當月總計金額）
  - 基本的收支預設類別（如：食、衣、住、行、育、樂）
- **Should Have (應該有)**
  - 週期性分析圖表（該月各類別支出比例的圓餅圖）
  - 預算超支提醒（設定該月總預算並在介面上顯示進度條/警告狀態）
- **Nice to Have (可以有)**
  - 自定義類別管理（讓使用者可自行擴充類別）
  - 資料匯出功能（如導出近半年明細為 CSV 檔案）
